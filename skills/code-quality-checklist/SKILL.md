# 代码质量检查清单

## 何时使用

- 代码审查时
- 提交代码前
- PR 检查时

## 审查清单

### 代码质量
- [ ] 代码简单易读
- [ ] 函数和变量命名良好
- [ ] 没有重复代码
- [ ] 单一职责原则
- [ ] 适当的抽象层次

### 安全性
- [ ] 没有暴露的密钥或 API 密钥
- [ ] 实现了输入验证
- [ ] 防止 XSS/SQL 注入
- [ ] 适当的错误信息（不泄露内部信息）

### 错误处理
- [ ] 适当的错误处理
- [ ] 边界情况处理
- [ ] 空值/undefined 检查

### 性能
- [ ] 没有 N+1 查询
- [ ] 适当的缓存
- [ ] 避免不必要的重渲染

### 测试
- [ ] 良好的测试覆盖
- [ ] 测试边界情况
- [ ] 测试错误路径

## 反馈优先级

| 优先级 | 类型 | 说明 |
|--------|------|------|
| P0 | 关键 | 必须修复（安全、崩溃） |
| P1 | 警告 | 应该修复（性能、逻辑） |
| P2 | 建议 | 考虑改进（风格、可读性） |
