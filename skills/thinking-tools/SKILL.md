# 思维工具箱

当你卡住、困惑或需要创新突破时，使用这些工具。

## 派遣器：选择正确工具

| 症状 | 工具 |
|------|------|
| 复杂度螺旋，越想越乱 | 简化级联 |
| 需要创新，找不到新角度 | 概念碰撞 |
| 被假设困住，感觉有隐藏约束 | 反转练习 |
| 规模不确定，不知道会怎么演变 | 规模测试 |

---

## 工具 1: 反转练习

**触发**: 感觉被隐藏假设困住

**方法**: 问 "如果反面为真会怎样？"

```
当前假设: "用户需要实时更新"
反转: "如果延迟更新反而更好？"
→ 发现: 批量更新减少服务器压力，用户体验可能更好
```

**示例应用**:
- "必须用缓存" → "如果不用缓存？" → 发现数据库已够快
- "需要微服务" → "如果单体更好？" → 发现当前规模不需要拆分
- "前端处理" → "如果后端处理？" → 发现可以减少前端复杂度

---

## 工具 2: 简化级联

**触发**: 问题太复杂，不知从何下手

**方法**: 问 "这是什么的特例？" 然后级联简化

```
复杂问题: "分布式事务一致性"
↓ 这是什么的特例？
"多节点状态同步"
↓ 这是什么的特例？
"两个节点状态同步"
↓ 这是什么的特例？
"单节点状态管理"
→ 从单节点开始设计，逐步扩展
```

**核心原则**:
- 先解决最简单版本
- 识别真正的复杂源
- 复杂度应该被证明需要，而非假设需要

## 工具 3: 规模测试

**触发**: 不确定方案在不同规模下是否可行

**方法**: 问 "如果放大 1000x？" 和 "如果缩小到 1？"

```
当前方案: "每次请求查询数据库"
放大 1000x: 1000 倍请求 → 数据库崩溃
缩小到 1: 单用户 → 完全没问题
→ 发现: 需要缓存层，但可以延迟添加
```

**规模维度**:
- 用户数: 1 → 1000 → 1M
- 数据量: 100 条 → 10M 条
- 并发: 单线程 → 高并发
- 时间: 1 天 → 1 年

## 工具 4: 概念碰撞

**触发**: 需要创新想法，常规思路走不通

**方法**: 强制两个不相关概念碰撞

```
问题: "如何提高代码审查效率？"
随机概念: "游戏化"
碰撞: 代码审查 × 游戏化
→ 想法: 审查积分系统、成就徽章、排行榜
```

**碰撞步骤**:
1. 明确定义问题
2. 随机选择一个领域（游戏/医疗/建筑/音乐...）
3. 强制找到连接点
4. 至少产生 3 个想法，再评估可行性

**常用碰撞领域**:
- 游戏机制
- 生物系统
- 城市规划
- 音乐结构
- 烹饪流程

## 使用原则

1. **先选工具再思考** - 不要漫无目的地想
2. **时间限制** - 每个工具 5 分钟，没结果就换
3. **写下来** - 思考过程可视化
4. **组合使用** - 一个工具的输出可以是另一个的输入

## 典型组合

```
复杂需求 → 简化级联（理清本质）→ 反转练习（挑战假设）→ 规模测试（验证方案）
创新任务 → 概念碰撞（产生想法）→ 简化级联（落地实现）
```
